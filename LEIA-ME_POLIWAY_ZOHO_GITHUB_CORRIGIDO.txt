POLIWAY — PACOTE CORRIGIDO V2 | LOGOS ANIMADAS

Senhor, este pacote corrige o problema das logos paradas.

O QUE FOI ALTERADO

1) O GIF animado recebeu um nome novo:
   logos_clientes_poliway_marquee_zoho_v2.gif

   Motivo: quando o arquivo antigo tem o mesmo nome, o navegador, o Zoho ou o GitHub podem continuar entregando uma versão antiga parada em cache.

2) O HTML de e-mail agora aponta para:
   https://raw.githubusercontent.com/ezrafchev/email-apresentacao-poliway/main/logos_clientes_poliway_marquee_zoho_v2.gif?v=20260702-02

3) Mantive também uma cópia com o nome antigo:
   logos_clientes_poliway_marquee.gif

   Assim, caso algum arquivo antigo ainda chame esse nome, ele também será animado.

4) O pacote possui um teste local direto:
   TESTE_RAPIDO_ABRIR_NO_NAVEGADOR.html

   Abra esse arquivo primeiro. Se a faixa mexer nele, o GIF está funcionando.

  Versão de teste local do e-mail usando o GIF da própria pasta.

  Cópia compatível com o nome antigo.

PASSO A PASSO CERTO PARA O ZOHO

1) Extraia esta pasta ZIP no seu computador.

2) Abra o arquivo:
   TESTE_RAPIDO_ABRIR_NO_NAVEGADOR.html

3) Confirme que as logos estão se movendo.

4) Entre no GitHub:
   https://github.com/ezrafchev/email-apresentacao-poliway/upload

5) Envie para a RAIZ do repositório estes arquivos:
   - index.html
   - email_poliway_zoho_completo_github.html
   - email_poliway_zoho_completo_github_codigo.txt
   - logos_clientes_poliway_marquee_zoho_v2.gif
   - logos_clientes_poliway_marquee.gif
   - logos_clientes_poliway_static.png

6) Faça o commit.

7) O repositório precisa estar PÚBLICO.
   Se estiver privado, o Zoho e os destinatários não conseguirão carregar o GIF.

8) Abra este link em guia anônima:
   https://raw.githubusercontent.com/ezrafchev/email-apresentacao-poliway/main/logos_clientes_poliway_marquee_zoho_v2.gif?v=20260702-02

9) Se o GIF abrir e se mover na guia anônima, o caminho está correto.

10) Abra:
    email_poliway_zoho_completo_github.html

11) Pressione Ctrl + A e Ctrl + C.

12) Cole no Zoho Mail.

13) Envie um teste real para outro e-mail.

IMPORTANTE

- O editor do Zoho pode mostrar o GIF parado durante a edição. O teste correto é enviar o e-mail e abrir a mensagem recebida.
- Outlook desktop costuma ser mais limitado com GIF; Gmail, Zoho Web e celular tendem a funcionar melhor.
- Em e-mail não existe garantia real de 60Hz como em site. Por isso, o pacote usa duas soluções:
  1) Web/GitHub: CSS com transform/translate3d, preparado para fluidez visual.
  2) E-mail/Zoho: GIF animado real, que é o método mais compatível.
